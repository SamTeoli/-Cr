## 기준

구조를 기준으로 한다.

## 구현 범위

- 이름, 등급, 유형, 역할과 호환 카드 종류를 원문대로 저장한다.
- 추가 호환 조건과 카드 화면 문구를 원문대로 저장한다.
- 각 효과의 발동 사건, 조건·대상, 처리 순서와 제한·표시를 분리해 저장한다.
- 각 인첸트의 v0.4 원본문서 주소를 보존한다.
- 같은 카드 중복은 E01~E08 모두 금지하고 다른 카드 장착은 허용한다.
- E01~E08을 `EnchantDatabase.asset`에 ID 순서로 연결한다.
- 이번 단계는 데이터 등록과 검증까지이며 실제 전투 효과 실행은 후속 단계에서 연결한다.
- C01~C12 카드 데이터는 변경하지 않는다.

## Unity에서 진행

1. 프로젝트를 연 뒤 컴파일이 끝날 때까지 기다린다.
2. `Have a Break > Card Data Tools`를 연다.
3. `Create/Rebuild Test Enchants E01-E08`을 한 번 누른다.
4. `Created or rebuilt E01-E08 and EnchantDatabase.`가 표시되는지 확인한다.
5. `Validate Test Enchants E01-E08`을 누른다.
6. `Test enchants E01-E08 passed.`가 표시되는지 확인한다.
7. Console에 빨간 컴파일 오류가 없는지 확인한다.
