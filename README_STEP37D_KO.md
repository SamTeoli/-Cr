# 37D — 적 행동 순서 계획

## 구현 내용

확정된 적 턴 규칙을 37B 행동 계획 생성 단계에 적용합니다.

- 적 처리 순서: 왼쪽 → 중앙 → 오른쪽
- 각 적의 행동 순서: 이동 → 공격 → 능력(버프·디버프 포함)
- 같은 적의 같은 종류 행동은 외부 시스템이 전달한 원래 순서를 유지
- 계획 생성 순간의 위치를 기준으로 순서를 고정
- 존재하지 않거나 쓰러졌거나 위치가 없는 적의 명령은 실행 전에 거부

적 행동의 종류와 수치, 대상 선택은 이 단계에서 만들지 않습니다. 외부 데이터가 만든
명령을 확정 규칙에 따라 정렬하며 런타임의 적 위치와 상태는 변경하지 않습니다.
검증 코드의 ID와 수치는 연결 확인용이며 게임 데이터가 아닙니다.

## Unity 검증

`Have a Break > Validate Runtime Enemy Turn Ordering`

성공 메시지:

`Battle runtime enemy turn ordering passed.`

검증은 섞인 명령의 좌→우·이동→공격→능력 정렬, 같은 종류 행동의 안정 순서,
런타임 상태 비변경과 잘못된 적 상태 거부를 확인합니다.
