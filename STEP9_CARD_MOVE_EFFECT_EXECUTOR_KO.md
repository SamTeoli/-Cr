# 9단계: 카드 이동 효과 실행기

## 기준

통합 기획서 v0.4의 9장 효과 처리와 10.3 공간 부족, 10.4 카드 이동 명령,
13.3 임시 카드의 소멸 대체 규칙을 기준으로 한다.

## 구현 범위

- 효과 대기열에서 우선순위가 가장 높은 명령을 한 개씩 꺼낸다.
- 명령이 참조하는 원본 사건이 실제 기록에 있는지 확인한다.
- 현재는 `Move` 명령만 실행하며 나머지는 `UnsupportedOperation`으로 구분한다.
- 대상 전투카드ID와 출발 영역을 최종 실행 직전에 다시 확인한다.
- 이동 성공 시에만 부모 사건과 출발·도착 영역을 가진 `CardMoved` 사건을 기록한다.
- 필드 공간 부족이나 대상 없음은 카드 위치를 바꾸지 않고 이동 사건도 만들지 않는다.
- 임시 카드가 묘지로 이동하려 하면 실제 도착 영역을 소멸로 대체한다.
- 드로우 더미에서 꺼내거나 되돌리는 효과는 위·아래 위치 규칙 전까지 실행하지 않는다.
- 내부 시작 패 교체 영역은 일반 효과의 목적지로 사용할 수 없다.

피해, 회복, 상태, 드로우와 마력 명령은 각 자료와 판정 순서를 구현하는 후속 단계에서
같은 실행기에 추가한다.

## Unity에서 확인

1. 프로젝트를 연 뒤 스크립트 컴파일이 끝날 때까지 기다린다.
2. `Have a Break > Card Data Tools`를 연다.
3. `Validate Card Move Effects`를 누른다.
4. `Card move effects passed.`가 표시되는지 확인한다.
5. Console에 빨간 컴파일 오류가 없는지 확인한다.
