# 15단계: 런 카드 인첸트 슬롯과 즉시 장착

## 기준

통합 기획서 v0.4의 14.1 기본 구조, 14.4 계산과 장착 순서, 14.5 인첸트 획득과
즉시 장착, 14.6 중복과 호환 규칙을 기준으로 한다.

## 구현 범위

- 인첸트 정의는 정의 ID, 이름, 등급, 호환 카드 종류와 같은 카드 중복 허용 여부를 가진다.
- 카드의 기본 인첸트 슬롯은 1칸이며 최대 3칸이다.
- 인첸트는 지정한 빈 슬롯에 즉시 장착한다.
- 장착 직전에 슬롯, 카드 종류 호환성과 같은 카드 중복 제한을 다시 검사한다.
- 중복 가능 속성이 없는 같은 인첸트는 한 카드에 두 번 장착할 수 없다.
- 같은 인첸트를 서로 다른 카드에 장착하는 것은 허용한다.
- 실제 장착 순서는 증가하는 번호로 기록하며 제거해도 남은 순서를 다시 매기지 않는다.
- 인첸트를 제거하면 해당 슬롯만 비고 다른 인첸트는 이동하지 않는다.
- 카드 종류가 비호환으로 바뀌면 인첸트를 제거하지 않고 비활성화한다.
- 카드 종류가 다시 호환되면 기존 슬롯에서 재활성화한다.
- 전투 중에는 인첸트 장착과 제거를 차단한다.
- 즉시 장착할 호환 카드와 빈 슬롯이 없으면 보상 후보로 사용할 수 없다.
- 실제 E01~E08 인첸트 정의와 효과 수치는 이 단계에서 임의 생성하지 않는다.
- 해제·변이 주문서의 소비와 인첸트 효과 계산은 후속 단계에서 연결한다.

## Unity에서 확인

1. 프로젝트를 연 뒤 스크립트 컴파일이 끝날 때까지 기다린다.
2. `Have a Break > Card Data Tools`를 연다.
3. `Validate Run Card Enchant Slots`를 누른다.
4. `Run card enchant slots passed.`가 표시되는지 확인한다.
5. Console에 빨간 컴파일 오류가 없는지 확인한다.
