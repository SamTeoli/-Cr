# 38A~38C — 플레이어 몬스터 위치와 적 공격 대상 선택 묶음

## 38A: 플레이어 몬스터 3칸 위치

- 플레이어 몬스터존을 왼쪽·중앙·오른쪽으로 구분합니다.
- 소환 시 위치를 명시할 수 있습니다.
- 기존 호출은 호환성을 위해 왼쪽부터 빈칸에 배치합니다.
- 몬스터가 파괴되면 등록 정보와 위치를 함께 정리합니다.

## 38B: 적 공격 대상 선택

- 정면 몬스터를 우선 선택합니다.
- 정면이 비었으면 가장 가까운 몬스터를 선택합니다.
- 양쪽 거리가 같으면 외부에서 전달된 난수 값으로 하나를 선택합니다.
- 필드 몬스터가 없으면 플레이어를 선택합니다.
- 선택 서비스는 호출할 때마다 현재 필드를 다시 읽어 연속공격의 대상 재탐색을 지원합니다.

## 38C: 전체 회귀검증 편입

기존 C01~C12 전체 런타임 회귀검증에 플레이어 몬스터 위치와 적 공격 대상 선택을
추가합니다. 검증용 ID와 수치는 게임 데이터가 아니며 카드·적 데이터는 변경하지 않습니다.

## Unity 통합 검증

다음 메뉴 하나만 실행합니다.

`Have a Break > Validate Full Battle Runtime C01-C12`

성공 메시지:

`Full battle runtime regression C01-C12, ordered enemy turns, and attack targeting passed.`

Console에는 `Player monster positions and enemy attack targeting` 성공 로그가 출력됩니다.
