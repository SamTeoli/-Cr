# 11단계: 몬스터 파괴 상태 기반 검사

## 기준

통합 기획서 v0.4의 9.4 상태 기반 검사, 10.4 카드 이동 명령, 11.3 피해 처리와
13.3 임시 카드 소멸 대체 규칙을 기준으로 한다.

## 구현 범위

- 피해 효과는 체력만 변경하고 몬스터를 즉시 영역 이동시키지 않는다.
- 원자적 효과가 끝난 뒤 상태 기반 검사에서 체력 0 이하 후보를 수집한다.
- 현재 플레이어 몬스터존의 안정된 필드 목록 순서로 후보를 처리한다.
- 일반 몬스터는 몬스터존에서 묘지로 이동한다.
- 임시 몬스터는 묘지 대신 소멸로 이동한다.
- 성공한 파괴만 `MonsterDestroyed` 사건으로 기록한다.
- 파괴 사건은 원인이 된 피해 등 부모 사건과 출발·도착 영역을 보유한다.
- 파괴된 몬스터는 전투 상태 레지스트리에서 제거한다.
- 같은 후보를 다시 검사해도 중복 파괴 사건을 만들지 않는다.
- 부모 사건이 존재하지 않으면 상태와 사건을 변경하지 않는다.

적 필드와 양쪽의 정확한 왼쪽·중앙·오른쪽 처리 순서는 적 배치·슬롯 구조가 추가될 때
같은 검사기에 연결한다. 만료 상태, 불법 위치와 승패 후보 검사는 후속 단계 범위다.

## Unity에서 확인

1. 프로젝트를 연 뒤 스크립트 컴파일이 끝날 때까지 기다린다.
2. `Have a Break > Card Data Tools`를 연다.
3. `Validate Monster Destruction Check`를 누른다.
4. `Monster destruction check passed.`가 표시되는지 확인한다.
5. Console에 빨간 컴파일 오류가 없는지 확인한다.
