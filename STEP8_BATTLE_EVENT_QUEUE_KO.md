# 8단계: 전투 사건과 효과 대기열

## 기준

통합 기획서 v0.4의 9장 효과 엔진, 사건과 대응 규칙을 기준으로 한다.

## 구현 범위

- 사건은 이미 발생한 사실이며 `BattleEventLog`에 순서대로 기록한다.
- 사건은 사건ID, 부모 사건ID, 종류, 원인, 출처 효과, 주체, 대상, 영역·수치 변화와
  난수 상태를 보유한다.
- 효과는 앞으로 처리할 명령이며 `BattleEffectQueue`에 등록한다.
- 처리 단계는 사전 수정 → 대응 → 본 효과 → 사후 유발 효과 순서다.
- 같은 대응 단계에서는 나중에 등록된 효과부터 역순으로 처리한다.
- 사후 유발 효과는 체계 필수, 출처 카드, 출처 인첸트, 같은 편, 반대편,
  결계·전역·상태, 정리 순서로 처리한다.
- 같은 사후 우선순위에서는 필수, 선택, 생성 순서로 처리한다.
- 같은 효과ID는 같은 사건에 기본적으로 한 번만 등록한다.
- 자기 효과가 만든 같은 종류의 사건에는 기본적으로 다시 등록하지 않는다.
- 반복을 허용하는 효과도 사건당 명시된 등록 상한을 넘을 수 없다.

실제 피해, 회복, 이동, 상태 적용과 선택 대응 화면은 이후 실행기 단계에서 이 대기열의
명령을 소비하도록 연결한다.

## Unity에서 확인

1. 프로젝트를 연 뒤 스크립트 컴파일이 끝날 때까지 기다린다.
2. `Have a Break > Card Data Tools`를 연다.
3. `Validate Battle Event Queue`를 누른다.
4. `Battle event queue passed.`가 표시되는지 확인한다.
5. Console에 빨간 컴파일 오류가 없는지 확인한다.
