# 37B — 적 행동 계획과 의도 데이터

## 구현 내용

- 외부 시스템이 만든 적 행동 명령을 한 턴의 계획으로 묶습니다.
- 원본 목록이 나중에 변경되어도 계획이 바뀌지 않도록 목록을 스냅샷으로 보관합니다.
- 이동·공격·능력 명령에서 화면 표시용 의도 정보를 함께 생성합니다.
- 명령별 필수 ID, 이동 칸 수, 공격 대상, 능력 문맥을 실행 전에 검증합니다.
- 검증된 계획은 37A 적 턴 지휘 시스템으로 그대로 전달해 실행합니다.

이 단계는 적의 행동 선택 규칙이나 실제 능력 수치를 만들지 않습니다. 외부 데이터가
결정한 명령을 안전하게 보관하고 UI가 읽을 수 있는 의도로 변환하는 공통 경계만
추가합니다. 검증 코드의 ID와 수치는 런타임 연결 확인용이며 게임 데이터가 아닙니다.

## Unity 검증

`Have a Break > Validate Runtime Enemy Turn Planning`

성공 메시지:

`Battle runtime enemy turn planning passed.`

검증은 계획 스냅샷, 이동·공격·능력 의도 변환, 잘못된 명령 거부,
계획에서 기존 적 턴 지휘 시스템으로 이어지는 실행을 확인합니다.
