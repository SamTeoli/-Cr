# 35B단계 — 런타임 카드 플레이·소환 사건 연결

카드 미리보기, 행동 단계 진입, 비용 지불, 카드 구역 이동, 카드 사용 사건 기록,
몬스터 등록과 소환 사건 기록, 행동 단계 복귀를 하나의 런타임 서비스로 연결합니다.

카드별 효과 해결 순서와 카드 데이터는 변경하지 않습니다.

## Unity 확인

1. ZIP의 `Assets` 폴더를 프로젝트에 덮어씁니다.
2. 컴파일 완료 후 Console 오류가 없는지 확인합니다.
3. `Have a Break > Validate Battle Runtime Card Play Events`를 실행합니다.
4. `Battle runtime card play events passed.`를 확인합니다.
