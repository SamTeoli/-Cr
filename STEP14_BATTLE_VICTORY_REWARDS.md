# 14단계: 전투 승리 보상 골드와 구성

## 기준

통합 기획서 v0.4의 3.2 전투 정산 뒤 보상 지급과 20.2~20.5 전투 등급별 보상을
기준으로 한다.

## 구현 범위

- 전투 종료 반영이 완료된 승리만 보상을 받을 수 있다.
- 패배 또는 아직 정산되지 않은 전투에는 보상을 지급하지 않는다.
- 일반 전투 골드는 20·25·30 중 하나를 동일 확률로 선택한다.
- 엘리트 골드는 40·45·50·55 중 하나를 동일 확률로 선택한다.
- 중간보스 골드는 60이다.
- 최종 보스는 런 전용 골드를 지급하지 않는다.
- 동일한 보상 시드는 동일한 골드 결과를 만든다.
- 보상 골드는 런 상태에 직접 한 번만 지급한다.
- 일반 전투는 인첸트 선택지 3개를 제공한다.
- 엘리트는 인첸트 선택지 3개, 희귀 이상 최소 1개와 소모아이템 1개를 제공한다.
- 중간보스는 희귀 이상 인첸트 선택지 3개를 제공한다.
- 최종 보스는 런 골드·인첸트·소모아이템 대신 영구 보상 흐름을 사용한다.
- 인첸트와 소모아이템의 실제 후보 ID 생성·장착·인벤토리 공간 처리는 후속 단계다.
- 최종 보스 카드팩 개봉권 수량은 기획서에서 미확정이므로 생성하지 않는다.

## Unity에서 확인

1. 프로젝트를 연 뒤 스크립트 컴파일이 끝날 때까지 기다린다.
2. `Have a Break > Card Data Tools`를 연다.
3. `Validate Battle Victory Rewards`를 누른다.
4. `Battle victory rewards passed.`가 표시되는지 확인한다.
5. Console에 빨간 컴파일 오류가 없는지 확인한다.
