# 2단계: 기능 수정과 콘텐츠 교체 준비

## 목적

전체 통합 테스트를 통과한 기능 검증용 프로젝트를 본 콘텐츠 제작이 가능한 구조로
전환한다. 시험 카드 C01~C12, 시험 인첸트 E01~E08, 시험 소모아이템과 시험 적은
회귀검사용으로 보존하되, 새 콘텐츠가 시험 ID나 시험 전용 실행기에 의존하지 않도록 한다.

## 현재 기준선

- Unity 전체 통합 테스트 통과
- Console 오류 0, 경고 0
- 새 런과 이어하기의 데이터 유실 방지 확인 절차 검증 완료
- 런타임 전투, 저장, 정산, 보상과 노드 진행 검증 완료

## 구현과 기획의 차이

### 유지할 기능 기반

- 카드 4종, 3등급과 카드 ID 계층
- 런스테이지와 배틀스테이지 분리
- 패, 덱, 묘지, 소멸과 필드 영역 처리
- 마력, 드로우, 턴, 적 행동과 상태이상 처리
- 인첸트 슬롯, 장착 순서와 호환성 검사
- 전투 정산, 보상 대기와 다음 노드 진행 잠금
- 전투 사이 런 저장과 전투 시작 체크포인트

### 수정이 필요한 기능

1. 런 노드는 현재 단계마다 즉석 선택지를 만든다. 실제 연결 경로와 선택한 경로의
   지속 데이터가 없다.
2. 3막 표시는 완료 노드 수에서 계산하며, 막별 콘텐츠 풀과 난이도 상승 데이터가 없다.
3. 상황 이벤트는 코드에 고정된 결과 3개뿐이며 데이터로 추가하거나 교체할 수 없다.
4. 상점 상품 수는 화면에서 만들어지지만 구매 상태를 상품 슬롯 데이터로 보존하지 않는다.
5. 임의 시점 전투 저장은 지원하지 않고 전투 시작 체크포인트로만 복원한다. 이 제한은
   현재 안전 규칙으로 유지하되 제품 범위에서 다시 결정해야 한다.
6. 런타임 화면은 기능 검증용 IMGUI이며 최종 UI가 아니다.

### 교체 준비가 필요한 콘텐츠

#### 카드

- C01~C12 에셋과 데이터베이스는 교체 가능하다.
- 실제 효과가 `TestCardEffectResolvers`와 시험 ID 어댑터에 연결되어 있어 새 효과를
  데이터만 추가해 실행할 수는 없다.
- 새 카드 효과 명세를 공용 효과 명령과 전용 해결기로 나누는 등록 경계가 필요하다.

#### 소모아이템

- `PrototypeConsumableCatalog`의 4종이 C# 코드에 하드코딩되어 있다.
- 정의 에셋과 데이터베이스가 없어 기획 데이터만으로 추가·가격 변경·보상 후보 변경을
  할 수 없다.
- 콘텐츠 교체의 첫 구조 작업으로 소모아이템 정의와 데이터베이스를 도입한다.

#### 인첸트

- E01~E08 정의 에셋과 데이터베이스는 존재한다.
- 실제 효과가 `TestEnchantEffectResolvers`에 모여 있어 시험 세트 교체에는 유리하지만,
  새 효과 등록 방식은 아직 시험 ID 중심이다.
- 기존 에셋 구조를 유지하면서 효과 실행 등록 경계를 일반화한다.

#### 적과 조우

- 적 정의는 시험 적 3종뿐이다.
- 일반, 엘리트, 중간보스와 보스 조우가 같은 시험 적 3종을 재사용한다.
- 등급별 적 목록과 조우 묶음 후보 풀이 없으며 런타임은 등급별 단일 조우 ID를 사용한다.
- 1~3마리, 빈칸, 중복, 고정·무작위 배치와 조우 시작 효과를 표현할 데이터 확장이 필요하다.

## 작업 순서

1. 소모아이템 정의 에셋과 데이터베이스 도입
2. 시험 소모아이템 4종을 에셋으로 이전하고 기존 회귀검사 유지
3. 등급별 조우 후보 풀과 결정적 무작위 선택 도입
4. 일반·엘리트·중간보스·보스 적 목록 분리
5. 카드와 인첸트 효과 등록 경계 일반화
6. 이벤트와 상점 상품 슬롯 데이터화
7. 기획 확정 데이터를 새 ID로 제작하고 시험 콘텐츠와 병행 검증
8. 최종 데이터 전환 후 시험 콘텐츠를 회귀검사용 별도 모듈로 격리

## 첫 구현 완료 조건

소모아이템 1차 구조 전환은 다음 조건을 모두 만족해야 한다.

- 소모아이템 ID, 이름, 효과 유형, 효과 수치와 상점 가격을 에셋에서 관리한다.
- 데이터베이스가 빈 ID와 중복 ID를 거부한다.
- 런, 상점, 전투 사용과 엘리트 보상이 데이터베이스 조회를 사용한다.
- 기존 시험 소모아이템 4종의 동작과 저장 ID가 바뀌지 않는다.
- 전체 통합 테스트가 계속 통과한다.
